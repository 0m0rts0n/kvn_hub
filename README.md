# kvn_hub

Минимальный репозиторий для GitHub Pages, где страница читает содержимое файла `data.txt`.
Проект рассчитан на `GitHub Free`, поэтому репозиторий для Pages должен быть публичным.

## Структура

- `index.html` - страница GitHub Pages;
- `script.js` - загрузка `data.txt` через `fetch()`;
- `styles.css` - оформление страницы;
- `data.txt` - текстовые данные, которые выводятся на странице.

## Публикация на GitHub Pages

1. Создайте публичный репозиторий `kvn_hub`.
2. Загрузите содержимое этой папки в ветку `main`.
3. В настройках репозитория откройте `Settings -> Pages`.
4. В `Build and deployment` выберите `GitHub Actions`.
5. После первого пуша workflow сам опубликует сайт.

После публикации страница будет доступна по адресу:

`https://0m0rts0n.github.io/kvn_hub/`

Важно:

- для личного аккаунта на `GitHub Free` Pages работает только из публичного репозитория;
- если нужен именно закрытый источник данных, GitHub Pages не подходит для прямого чтения `.txt` из приватного репозитория на клиенте;
- текущая конфигурация публикует только статические файлы из этого проекта.
